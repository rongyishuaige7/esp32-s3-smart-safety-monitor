这些额外对象都不导入公开候选。原始目录没有可继承的 Git 历史，因此公开仓从干净根目录建立。

## 纳入内容

- 当前 ESP-IDF 应用源码、PlatformIO 配置与公开 `sdkconfig.defaults`；
- 原始的传感器、执行器、OLED、雷达和本地状态页源码；
- 从源码推导的 BOM、GPIO 接口边界图、构建/状态/协议/安全文档；
- 新增的公开安全默认配置、无硬件门禁和测试。

## 发布前排除和净化

- 固定 SoftAP SSID `SafetyMonitor` 与密码：固定密码不进入公开候选。公开源码默认从 `local_config.example.h` 读取空值；只有本地、被忽略的 `local_config.h` 才能启用状态页；
- `sdkconfig.esp32-s3-devkitc-1`、`.pio/`、`build/`、生成的 `web_index_embed.c`、编辑器文件和历史 ZIP；
- 原始演示步骤中“对着传感器喷丁烷/点火”“水泵演示”等危险操作文字；公开文档改为风险与受控低压测试边界；
- 本机路径、私网/热点凭据、实物照片、视频、PCB/EDA、构建二进制和不可审计的历史构建结果。

## 公开硬化

- `platformio.ini` 固定 `espressif32@6.13.0` 与 `esp32-s3-devkitc-1`；
- HTML 嵌入 C 文件改为只在构建目录生成，避免提交大体积生成物；
- 启动网络前检查本地配置，未配置时不启动 SoftAP 或 HTTP；
- 日志不回显 SSID、密码或固定本地 IP；
- 删除浏览器预览时生成伪造传感器数据的 DEMO 模式；预览只显示“未连接设备”；
- 移除页面对 Google Fonts 的远程 `@import`，避免隔离 AP 页面隐式访问第三方字体服务；
- 将网页阈值与 `app_config.h` 同步，并将 UI 文案改为原始样本/教学逻辑而非真实安全结论；
- 通过 README、硬件、安全和协议文件明确电气、网络、误报/漏报与高后果边界。

这些措施不证明原始硬件可靠、安全、无误报/漏报或适于任何现实安全用途。
