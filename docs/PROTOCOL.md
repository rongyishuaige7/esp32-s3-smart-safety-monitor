# 本地状态页协议边界

## 前置条件

仅在用户自行创建本地、被 Git 忽略的 `main/local_config.h`，且 SSID 非空、密码至少 8 位时，固件才启动 SoftAP 与 HTTP 服务。公开默认配置不会启动该服务。

## 路由

| 路由 | 方法 | 用途 | 安全边界 |
| :-- | :-- | :-- | :-- |
| `/` | GET | 返回内嵌 HTML 本地状态页 | 无认证、无 TLS；只可用于隔离可信本地网络 |
| `/api/status` | GET / SSE | 每秒发送软件快照 | 无认证、无 TLS；不是执行器反馈、环境安全、报警送达或外部服务状态 |
| 常见图标路径 | GET | 返回 204 以减少日志噪声 | 不提供实际图标内容 |

`/api/status` 的字段来自 `g_app_state`：`mq2`、`mq5`、`flame`、`dist`、`smoke_level`、`gas_danger`、`buzzer`、`led`、`pump`、`fan`、`window` 与 `state`。其中：

- `mq2`、`mq5` 是 ADC 原始值，不是浓度或校准读数；
- `state` 是代码分支的 UI 状态，不是安全风险分类；
- `buzzer`、`pump`、`fan`、`window` 是软件意图/状态字段，不是传感器或执行器真实动作反馈；
- 路由没有身份、会话、访问控制、TLS、请求签名、速率限制或审计机制。

请不要将页面、串口或网络抓包中的任何“连接”“正常”“报警”字样用于安全、消防、燃气、生产或人身风险结论。
