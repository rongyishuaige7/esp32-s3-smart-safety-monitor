# 安全、隐私与高后果边界

这是一个 ESP32-S3 低压软硬件学习原型，不是经过认证或适合部署的安全系统。

## 生命安全与物理风险

- MQ-2/MQ-5、火焰模块和超声波模块可能误报、漏报、漂移、受环境光/温湿度/预热/安装方向影响或完全失效。代码的原始 ADC 阈值不是浓度、距离、火焰或风险判定。
- 水泵、风扇、舵机和继电器/驱动周边可能产生浪涌、反电动势、夹伤、进水、短路、过热或不可预测动作。启动自检会顺序驱动 LED、蜂鸣器、两路舵机、风扇和水泵；第一次上电或改线时必须断开执行负载或使用受控低压测试条件。
- 绝不把此原型作为火灾报警、燃气泄漏报警、自动灭火、通风决策、家庭安防、人员疏散、设备互锁、无人值守保护或生产控制依据。
- 未公开电池、充电、保险丝、浪涌、过流、隔离、EMC、外壳、防水、防火、失电、冗余、故障自检、监管或认证设计。

## 本地网络与凭据

- 公开仓没有固定 Wi-Fi SSID 或密码。真实热点信息只能放在本地、被忽略的 `main/local_config.h`，不能提交、截图、打印或写入 Issue。
- 未配置 `local_config.h` 时，Wi-Fi/HTTP 服务不会启动；这不是运行故障，而是公开安全默认值。
- 配置后，SoftAP、`/` 和 `/api/status` 都是**无认证、无 TLS、无设备身份、无会话和无访问控制**的本地服务。它只能短时运行在隔离可信网络，严禁暴露公网。
- SSE 页面显示的是固件当前发送的软件状态；没有执行器反馈、传感器校准、远程告警送达或外部服务健康语义。不要把“已连接”“NORMAL”等页面文本视作安全或真实环境结论。

## 报告问题

公开 Issue 中不要粘贴 SSID、密码、私网 IP、MAC、串口日志、照片 EXIF/GPS、真实地点、设备标识或复现危险物理行为的详细操作。安全问题请先通过 GitHub 账户主页的公开联系方式做最小化说明；在收到确认前不要披露可被滥用的凭据或攻击细节。
